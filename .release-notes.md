## v0.1.1 — first npm release

Pier is installable from npm:

```bash
npm i -g mcpier
pier login http://pier.homelab:8420 --token <device-token>
pier install brainstorm-mcp --location remote --sync claude-code
```

### Highlights since initial scaffold

- **PIER_MASTER_KEY_FILE** — file-backed at-rest encryption key, works with Docker secrets, K8s projected volumes, Vault Agent renders
- **`pier backup` + `pier restore`** — JSON bundle round-trip with pre-flight master-key-match validation (rejects mismatched keys *before* touching disk)
- **PIER_SPAWN_MEMORY_MB=512** — per-subprocess memory caps via `prlimit` on Linux; runaway `npx`/`uvx` MCPs can't OOM the host
- **`pier install <name>`** + **`pier install-git <url>`** — one-command MCP installs from subscribed catalogs or any git repo with a `pier.yaml`
- **Three one-command deploys** — Docker run, Docker Compose, and Kubernetes (`kubectl apply -f`) all pull `ghcr.io/spranab/mcpier:latest`
- **Multi-catalog browse + search** — subscribe to any `catalog.json` URL; live search against the [official MCP Registry](https://registry.modelcontextprotocol.io) surfaces namespace-authenticated entries (e.g. `com.stripe/mcp`, `com.microsoft/azure`)
- **npm provenance** — published tarball carries an attestation linking it to this commit on a GitHub-Actions runner

### Install summary

| Runtime | One command |
|---|---|
| Docker Compose | `docker compose -f https://raw.githubusercontent.com/spranab/mcpier/main/deploy/compose/install.yml up -d` |
| Docker run | `docker run -d --name pier -p 8420:8420 -v pier-data:/data -e PIER_MASTER_KEY=... -e PIER_TOKENS=... ghcr.io/spranab/mcpier:latest` |
| Kubernetes | `kubectl apply -f https://raw.githubusercontent.com/spranab/mcpier/main/deploy/kubernetes/install.yaml` |
